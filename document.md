## Ansible とは

サーバーやネットワーク機器、クラウド上のリソースに対して行う構築・設定変更・アプリ配布・運用作業を自動化するための「構成管理／自動化ツール」

- メリット

| 性質     | 内容                                                                 |
| :------- | :------------------------------------------------------------------- |
| 再現性   | 誰がやっても同じ結果になる（手順漏れ・ミスを減らす）                 |
| 冪等性   | 同じ Playbook を何度実行しても、必要な差分だけ適用される設計にできる |
| 可視化   | 作業内容が YAML として残り、レビューや変更履歴管理（Git）に向く      |
| 一括適用 | 複数台（10 台、100 台）に同じ設定をまとめて適用できる                |

- 構成要素

| 構成名       | 役割                                       |
| :----------- | :----------------------------------------- |
| インベントリ | どのサーバーを管理するかを定義するファイル |
| プレイブック | 何をするかを定義する YAML ファイル         |
| ロール       | 決まったディレクトリ構造を持つ部品         |
| モジュール   | 実際の作業を行う部品。                     |
| 変数         |                                            |
